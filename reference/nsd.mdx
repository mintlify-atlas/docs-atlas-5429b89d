# nsd

Name Server Daemon (NSD) - Authoritative DNS nameserver

## Synopsis

```bash
nsd [-4] [-6] [-a ip-address[@port]] [-c configfile] [-d] [-h] 
    [-i identity] [-I nsid] [-l logfile] [-N server-count] 
    [-n noncurrent-tcp-count] [-P pidfile] [-p port] 
    [-s seconds] [-t chrootdir] [-u username] [-V level] [-v]
```

## Description

**NSD** is a complete implementation of an authoritative DNS nameserver. Upon startup, NSD reads the configuration file, daemonizes itself, and answers queries on port 53 (or a different port specified with the `-p` option).

By default, NSD binds to all local interfaces available. Use the `-a` option to specify a single particular interface address to be bound. If this option is given more than once, NSD will bind its UDP and TCP sockets to all the specified IP addresses separately.

Normally NSD should be started with the `nsd-control start` command invoked from a startup script at the operating system startup.

## Options

All options can be specified in the config file (see `-c` argument), except for `-v` and `-h`. If options are specified on the command line, they take precedence over the options in the config file.

<ParamField path="-4" type="flag">
  Only listen to IPv4 connections.
</ParamField>

<ParamField path="-6" type="flag">
  Only listen to IPv6 connections.
</ParamField>

<ParamField path="-a" type="string">
  Listen to the specified IP address. The IP address must be specified in numeric format (using the standard IPv4 or IPv6 notation). Optionally, a port number can be given.
  
  This flag can be specified multiple times to listen to multiple IP addresses. If this flag is not specified, NSD listens to the wildcard interface.
  
  ```bash
  nsd -a 192.0.2.1
  nsd -a 192.0.2.1@5353
  nsd -a 2001:db8::1@5353
  ```
</ParamField>

<ParamField path="-c" type="string">
  Read specified config file instead of the default. For format description see [nsd.conf(5)](/reference/nsd-conf).
  
  ```bash
  nsd -c /etc/nsd/nsd.conf
  ```
</ParamField>

<ParamField path="-d" type="flag">
  Do not fork, stay in the foreground. This is useful for debugging or when running under a process supervisor.
</ParamField>

<ParamField path="-h" type="flag">
  Print help information and exit.
</ParamField>

<ParamField path="-i" type="string">
  Return the specified identity when asked for `CH TXT ID.SERVER`. This option is used to determine which server is answering the queries when they are anycast. The default is the name returned by `gethostname(3)`.
  
  ```bash
  nsd -i "ns1.example.com"
  ```
</ParamField>

<ParamField path="-I" type="string">
  Add the specified NSID to the EDNS section of the answer when queried with an NSID EDNS enabled packet. Can be specified as a sequence of hex characters or with `ascii_` prefix and then an ASCII string.
  
  ```bash
  nsd -I "6e73312e6578616d706c652e636f6d"
  nsd -I "ascii_ns1.example.com"
  ```
</ParamField>

<ParamField path="-l" type="string">
  Log messages to the specified log file. The default is to log to stderr and syslog. If a `zonesdir:` is specified in the config file, this path can be relative to that directory.
  
  ```bash
  nsd -l /var/log/nsd.log
  ```
</ParamField>

<ParamField path="-N" type="number">
  Start this number of NSD servers. The default is 1. Starting more than a single server is only useful on machines with multiple CPUs and/or network adapters.
  
  ```bash
  nsd -N 4
  ```
</ParamField>

<ParamField path="-n" type="number">
  The maximum number of concurrent TCP connections that can be handled by each server. The default is 100.
  
  ```bash
  nsd -n 200
  ```
</ParamField>

<ParamField path="-P" type="string">
  Use the specified PID file instead of the platform-specific default. If a `zonesdir:` is specified in the config file, this path can be relative to that directory.
  
  ```bash
  nsd -P /var/run/nsd/nsd.pid
  ```
</ParamField>

<ParamField path="-p" type="number">
  Answer the queries on the specified port. Normally this is port 53.
  
  ```bash
  nsd -p 5353
  ```
</ParamField>

<ParamField path="-s" type="number">
  Produce statistics dump every N seconds. This is equal to sending `SIGUSR1` to the daemon periodically.
  
  ```bash
  nsd -s 3600
  ```
</ParamField>

<ParamField path="-t" type="string">
  Specifies a directory to chroot to upon startup. This option requires you to ensure that an appropriate `syslogd(8)` socket (e.g., `chrootdir/dev/log`) is available, otherwise NSD won't produce any log output.
  
  ```bash
  nsd -t /var/chroot/nsd
  ```
</ParamField>

<ParamField path="-u" type="string">
  Drop user and group privileges to those of the specified username after binding the socket. The username must be one of: `username`, `id`, or `id.gid`.
  
  ```bash
  nsd -u nsd
  nsd -u 80
  nsd -u 80.80
  ```
</ParamField>

<ParamField path="-V" type="number">
  This value specifies the verbosity level for (non-debug) logging. Default is 0.
  
  - **0**: Warnings and errors only
  - **1**: Additionally prints successful notifies, zone transfers, and failures
  - **2**: Additionally prints soft errors like connection resets
  
  ```bash
  nsd -V 2
  ```
</ParamField>

<ParamField path="-v" type="flag">
  Print the version number of NSD to standard error and exit.
</ParamField>

## Signals

NSD reacts to the following signals:

<ParamField path="SIGTERM">
  Stop answering queries, shutdown, and exit normally.
</ParamField>

<ParamField path="SIGHUP">
  Reopen log file (assists rotation) and optionally update TSIG keys and zones.
</ParamField>

<ParamField path="SIGUSR1">
  Dump BIND8-style statistics into the log.
</ParamField>

## Files

- **PID file**: Default location for the process ID file
- **Config file**: Default NSD configuration file (see [nsd.conf](/reference/nsd-conf))

## Diagnostics

NSD will log all problems via the standard `syslog(8)` daemon facility, unless the `-d` option is specified.
